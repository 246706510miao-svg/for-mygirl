# 接口定义说明

## 1. 总体约束

当前接口服务于 MVP。

技术边界：

- 前端：TypeScript Web
- 用户端：iPhone 移动 Web 或 PWA
- 管理员端：PC Web
- 后端：Java Spring Boot

外部能力边界：

- 前端不直接调用大模型
- 前端不直接调用语音识别服务
- 前端不直接调用飞书 MCP
- 所有外部服务调用统一经过后端

MVP 暂不做：

- 上传背景图
- 上传照片
- 长期保存语音音频
- 服务器文件存储
- 复杂多用户权限系统
- 管理员统计图表

## 2. 用户端接口

### 2.1 获取首页展示

```http
GET /api/user/home?date=2026-05-27
```

响应：

```json
{
  "date": "2026-05-27",
  "dailyContent": {
    "homeText": "今天记录的时候可以重点关注身体状态。",
    "theme": "warm",
    "backgroundPreset": "sunset",
    "recordGuide": "今天发生了什么？"
  },
  "recentRecords": [
    {
      "recordId": "record_001",
      "title": "2026-05-27 日常记录",
      "summary": "上午工作推进较慢，下午完成登录功能调试。",
      "score": 7,
      "status": "success"
    }
  ]
}
```

说明：用户端首页不直接读取飞书，数据来自 DAILY_CONTENT、RECORD_DISPLAY、DAILY_RECORD 状态。

### 2.2 创建记录会话

```http
POST /api/record-sessions
```

请求：

```json
{
  "recordDate": "2026-05-27"
}
```

响应：

```json
{
  "sessionId": "session_001",
  "status": "editing"
}
```

### 2.3 提交文本或修改指令

```http
POST /api/record-sessions/{sessionId}/messages
```

文本请求：

```json
{
  "clientMessageId": "msg_client_001",
  "content": "今天上午工作有点卡，下午把登录功能调通了",
  "messageType": "text"
}
```

修改请求：

```json
{
  "clientMessageId": "msg_client_002",
  "content": "把上午工作有点卡改成上午推进得比较慢",
  "messageType": "modify"
}
```

响应：

```json
{
  "sessionId": "session_001",
  "messageId": "msg_001",
  "draftId": "draft_003",
  "draftVersion": 3,
  "previewText": "今天上午推进较慢，下午完成了登录功能调试。",
  "draftJson": {
    "summary": "上午推进较慢，下午完成登录功能调试",
    "tags": ["工作", "开发"],
    "score": 7
  },
  "chatReply": "我已经把表述调整得更温和了，你可以再确认一下。"
}
```

说明：普通文本和修改指令共用此接口；后端保存 RECORD_MESSAGE；AI 输出的新草稿保存为 RECORD_DRAFT；返回必须包含 chatReply。

### 2.4 提交语音并生成草稿

```http
POST /api/record-sessions/{sessionId}/voice-messages
Content-Type: multipart/form-data
```

请求字段：

| 字段 | 说明 |
|---|---|
| audio | 音频文件 |
| clientMessageId | 前端生成的消息 ID |
| language | 可选，例如 zh-CN |

响应：

```json
{
  "sessionId": "session_001",
  "messageId": "msg_002",
  "asrText": "今天晚上去健身了，但是状态一般",
  "draftId": "draft_004",
  "draftVersion": 4,
  "previewText": "今天晚上进行了健身，但整体状态一般。",
  "chatReply": "我已经把这段健身记录补充进草稿了。"
}
```

说明：前端只负责录音和上传，后端调用 ASR，MVP 不长期保存音频文件。

### 2.5 查询当前记录会话

```http
GET /api/record-sessions/{sessionId}
```

响应：

```json
{
  "sessionId": "session_001",
  "status": "previewing",
  "currentDraftId": "draft_004",
  "messages": [
    {
      "sender": "user",
      "inputType": "text",
      "content": "今天上午工作有点卡"
    },
    {
      "sender": "user",
      "inputType": "voice",
      "content": "今天晚上去健身了，但是状态一般"
    }
  ],
  "currentDraft": {
    "draftId": "draft_004",
    "previewText": "今天上午工作推进较慢，下午完成登录功能调试，晚上进行了健身但状态一般。"
  }
}
```

### 2.6 确认写入飞书

```http
POST /api/record-sessions/{sessionId}/confirm
```

请求：

```json
{
  "draftId": "draft_004",
  "clientConfirmId": "confirm_001"
}
```

成功响应：

```json
{
  "recordId": "record_001",
  "sessionId": "session_001",
  "syncStatus": "success",
  "chatReply": "已经记录好了，并同步到了飞书。",
  "displayRecord": {
    "title": "2026-05-27 日常记录",
    "summary": "上午工作推进较慢，下午完成登录功能调试，晚上健身状态一般。",
    "score": 7,
    "status": "success"
  }
}
```

飞书同步失败响应：

```json
{
  "recordId": "record_001",
  "sessionId": "session_001",
  "syncStatus": "failed",
  "chatReply": "内容已经在本地保存，但同步飞书失败了，我会保留待重试记录。",
  "displayRecord": {
    "title": "2026-05-27 日常记录",
    "summary": "上午工作推进较慢，下午完成登录功能调试，晚上健身状态一般。",
    "score": 7,
    "status": "sync_failed"
  }
}
```

强制约束：

- 必须使用 clientConfirmId 做幂等控制
- 重复提交不能生成多条 DAILY_RECORD
- 确认后必须锁定 RECORD_SESSION
- AI Payload 修正次数最多 2 次
- 超过次数后标记为 blocked 或 sync_failed
- 即使飞书失败，也必须保存 DAILY_RECORD 和 FEISHU_SYNC
- 响应必须返回 chatReply 和 displayRecord

### 2.7 放弃记录会话

```http
POST /api/record-sessions/{sessionId}/cancel
```

响应：

```json
{
  "sessionId": "session_001",
  "status": "cancelled"
}
```

### 3.5 处理异常记录

```http
POST /api/admin/records/{recordId}/retry-sync
```

请求：

```json
{
  "retryMode": "auto"
}
```

成功响应：

```json
{
  "recordId": "record_001",
  "syncStatus": "success",
  "message": "重试成功，已同步飞书"
}
```

失败响应：

```json
{
  "recordId": "record_001",
  "syncStatus": "failed",
  "message": "重试失败，已更新失败原因",
  "errorMessage": "飞书字段类型不匹配"
}
```

强制约束：一次 retry-sync 请求中，AI 修正 Payload 最多 1 到 2 次；失败后保存错误原因；不允许无限循环重试。

### 3.6 更新用户端展示内容

```http
PATCH /api/admin/records/{recordId}/display
```

请求：

```json
{
  "adminContentJson": {
    "note": "今天整体完成度不错，明天注意恢复。",
    "highlight": "登录功能调通"
  }
}
```

响应：

```json
{
  "recordId": "record_001",
  "success": true,
  "message": "展示内容已更新"
}
```

说明：更新 RECORD_DISPLAY，不直接改飞书内容。
