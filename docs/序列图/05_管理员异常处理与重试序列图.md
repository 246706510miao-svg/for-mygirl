# 管理员异常处理与重试序列图

本图覆盖管理员查看记录异常、重试飞书同步，以及手动修正用户端展示内容的链路。

```mermaid
sequenceDiagram
    autonumber
    actor Admin as 管理员
    participant FE as 管理员端前端
    participant API as SpringBoot API
    participant BizDB as MySQL 业务库
    participant ThirdAPI as third Workflow API
    participant Redis as Redis
    participant Worker as third worker
    participant ThirdDB as MySQL third 库
    participant AI as OpenAI API
    participant Feishu as 飞书 OpenAPI

    Admin->>FE: 打开用户最近记录页
    FE->>API: GET /api/admin/records?onlyAbnormal=true&page&pageSize
    API->>BizDB: 查询 DAILY_RECORD + FEISHU_SYNC + RECORD_DISPLAY
    BizDB-->>API: 异常记录列表
    API-->>FE: items, page, total
    FE-->>Admin: 展示同步失败和 blocked 记录

    Admin->>FE: 点击详情
    FE->>API: GET /api/admin/records/{recordId}
    API->>BizDB: 查询 DAILY_RECORD、RECORD_DRAFT、RECORD_DISPLAY、最新 FEISHU_SYNC
    BizDB-->>API: 最终正文、结构化结果、payload、错误原因
    API-->>FE: record detail
    FE-->>Admin: 展示记录详情和失败原因

    alt 重试飞书同步
        Admin->>FE: 点击再次尝试提交
        FE->>API: POST /api/admin/records/{recordId}/retry-sync<br/>mode
        API->>BizDB: 校验记录状态、读取 payload 或准备重生成上下文

        alt reuse_payload
            API->>ThirdAPI: POST /workflows/invoke<br/>提交已保存 payload 重试
        else regenerate_payload
            API->>ThirdAPI: POST /workflows/invoke<br/>提交最终记录和 Schema 重新生成 payload
        end

        ThirdAPI->>ThirdDB: 保存 workflow session / plan
        ThirdAPI->>Redis: 投递 workflow 队列
        Worker->>Redis: 消费 workflow

        opt 重新生成 payload
            Worker->>AI: 重新生成并修正 payload
            AI-->>Worker: payload
        end

        Worker->>Feishu: 写入飞书
        Feishu-->>Worker: feishuRefId 或错误
        Worker->>ThirdDB: 保存 Tool 结果

        API->>ThirdAPI: GET /workflows/{thirdSessionId}
        ThirdAPI-->>API: workflow 结果
        API->>BizDB: 新增 FEISHU_SYNC，更新 DAILY_RECORD 和 RECORD_DISPLAY
        API-->>FE: retry result
        FE-->>Admin: 展示最新同步状态
    else 手动修正展示
        Admin->>FE: 修改摘要、评分或反馈
        FE->>API: PATCH /api/admin/records/{recordId}/display
        API->>BizDB: 更新 RECORD_DISPLAY
        BizDB-->>API: display
        API-->>FE: display
        FE-->>Admin: 展示修正后的用户端展示内容
    end
```

## 前后端契约重点

- 管理员详情页必须能看到飞书同步状态、payload 快照和错误原因。
- 重试同步建议新增 `FEISHU_SYNC` 记录，保留历史失败原因。
- `PATCH /api/admin/records/{recordId}/display` 只修改本地用户端展示，不直接修改飞书。
- 重试次数必须有限制，不能无限自动重试。
