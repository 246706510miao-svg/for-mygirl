# 确认写入与飞书同步序列图

本图覆盖用户点击“确认写入”后的主链路：SpringBoot 校验业务库状态并锁定草稿，third 判断飞书 payload/schema 并调用飞书，SpringBoot 再保存本地正式记录和用户端展示数据。

```mermaid
sequenceDiagram
    autonumber
    actor User as 用户
    participant FE as 用户端前端
    participant API as SpringBoot API
    participant BizDB as MySQL 业务库
    participant ThirdAPI as third Workflow API
    participant Redis as Redis
    participant Worker as third worker
    participant ThirdDB as MySQL third 库
    participant AI as OpenAI API
    participant Feishu as 飞书 OpenAPI

    User->>FE: 点击确认写入
    FE->>API: POST /api/record-sessions/{sessionId}/confirm<br/>clientConfirmId, draftId
    API->>BizDB: 校验 session、draftId 和 clientConfirmId 幂等

    alt 重复确认
        BizDB-->>API: 返回首次确认结果
        API-->>FE: record, feishuSync, display
        FE-->>User: 展示首次确认结果
    else 首次确认
        API->>BizDB: 锁定 RECORD_DRAFT(status=confirmed)
        API->>BizDB: 更新 RECORD_SESSION(status=confirming)
        API->>ThirdAPI: POST /workflows/invoke<br/>提交最终草稿和飞书写入意图
        ThirdAPI->>ThirdDB: 保存 workflow session / plan
        ThirdAPI->>Redis: 投递 workflow 队列
        ThirdAPI-->>API: thirdSessionId, status=queued

        Worker->>Redis: 消费 workflow
        Worker->>AI: 生成结构化记录和飞书 MCP payload
        AI-->>Worker: payload 候选
        Worker->>Worker: 校验飞书目标、字段、类型、必填、权限、内容安全

        alt 需要 AI 修正
            Worker->>AI: 带校验错误重新生成 payload
            AI-->>Worker: 修正后的 payload
            Worker->>Worker: 再次校验
        else 安全或权限异常
            Worker->>ThirdDB: 保存 blocked artifact
        end

        opt third 写入确认门
            ThirdAPI-->>API: status=waiting_user, confirmation
            API->>ThirdAPI: POST /workflows/{thirdSessionId}/resume<br/>approved=true
        end

        Worker->>Feishu: 写入多维表格或文档
        Feishu-->>Worker: feishuRefId 或错误
        Worker->>ThirdDB: 保存 payload、Tool 结果和错误信息

        loop 查询 workflow 结果
            API->>ThirdAPI: GET /workflows/{thirdSessionId}
            ThirdAPI->>ThirdDB: 读取状态和 artifact
            ThirdAPI-->>API: running / success / failed
        end

        alt 飞书写入成功
            API->>BizDB: 保存 DAILY_RECORD(status=success)
            API->>BizDB: 保存 FEISHU_SYNC(sync_status=success, payload_json, feishu_ref_id)
            API->>BizDB: 保存 RECORD_DISPLAY(display_status=success)
        else 飞书写入失败
            API->>BizDB: 保存 DAILY_RECORD(status=sync_failed)
            API->>BizDB: 保存 FEISHU_SYNC(sync_status=failed, payload_json, error_message)
            API->>BizDB: 保存 RECORD_DISPLAY(display_status=sync_failed)
        else payload 被阻断
            API->>BizDB: 保存 DAILY_RECORD(status=blocked)
            API->>BizDB: 保存 FEISHU_SYNC(sync_status=failed, error_message) 或阻断记录
            API->>BizDB: 保存 RECORD_DISPLAY(display_status=blocked)
        end

        API->>BizDB: 更新 RECORD_SESSION(status=confirmed)
        API->>BizDB: 保存系统消息 RECORD_MESSAGE
        API-->>FE: record, feishuSync, display, replyMessage
        FE-->>User: 展示确认写入结果卡片
    end
```

## 前后端契约重点

- `clientConfirmId` 是确认写入的幂等键。
- 用户确认时必须锁定明确的 `draftId`，不能确认已经被替换的草稿。
- SpringBoot 负责 `session/draftId/clientConfirmId` 和业务表落库校验；飞书 payload/schema 必须由 `third` 校验后才能调用飞书。
- 飞书失败时接口仍可返回业务成功，因为本地正式记录和展示数据已经保存。
- `sync_failed` 和 `blocked` 都要能在用户最近记录页和管理员记录页展示。
