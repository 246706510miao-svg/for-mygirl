# 记录草稿生成序列图

本图覆盖记录对话页中“用户输入文本或语音 -> AI 维护草稿 -> 前端展示草稿卡片”的调用顺序。新增、修改、删除和补充说明都走同一条消息链路，由 AI 根据上下文判断意图。

```mermaid
sequenceDiagram
    autonumber
    actor User as 用户
    participant FE as 用户端前端
    participant API as SpringBoot API
    participant ASR as ASR 服务
    participant BizDB as MySQL 业务库
    participant ThirdAPI as third Workflow API
    participant Redis as Redis
    participant Worker as third worker
    participant ThirdDB as MySQL third 库
    participant AI as OpenAI API

    User->>FE: 进入记录对话页
    FE->>API: POST /api/record-sessions
    API->>BizDB: 创建 RECORD_SESSION(status=editing)
    BizDB-->>API: session
    API-->>FE: session

    alt 文本输入或修改指令
        User->>FE: 输入文本
        FE->>API: POST /api/record-sessions/{sessionId}/messages<br/>clientMessageId, content
    else 语音输入
        User->>FE: 录音
        FE->>API: POST /api/record-sessions/{sessionId}/voice-messages<br/>clientMessageId, audio
        API->>ASR: 识别音频
        ASR-->>API: asrText
    end

    API->>BizDB: 校验 session 状态和 clientMessageId 幂等
    API->>BizDB: 保存 RECORD_MESSAGE
    API->>BizDB: 读取历史 RECORD_MESSAGE 和当前 RECORD_DRAFT
    API->>ThirdAPI: POST /workflows/invoke<br/>提交草稿生成上下文
    ThirdAPI->>ThirdDB: 保存 workflow session / plan
    ThirdAPI->>Redis: 投递 workflow 队列
    ThirdAPI-->>API: thirdSessionId, status=queued

    Worker->>Redis: 消费 workflow
    Worker->>ThirdDB: 读取 plan、prompt、tool registry
    Worker->>AI: 根据历史消息和当前草稿生成新草稿
    AI-->>Worker: draft_json, preview_text, tags, score
    Worker->>ThirdDB: 保存 artifact

    loop 查询 workflow 结果
        API->>ThirdAPI: GET /workflows/{thirdSessionId}
        ThirdAPI->>ThirdDB: 读取状态和 artifact
        ThirdAPI-->>API: running 或 success
    end

    API->>BizDB: 旧 RECORD_DRAFT 标记 replaced
    API->>BizDB: 新增 RECORD_DRAFT(status=active)
    API->>BizDB: 更新 RECORD_SESSION(status=previewing, current_draft_id)
    API->>BizDB: 保存 AI 回复 RECORD_MESSAGE
    API-->>FE: session, userMessage, aiMessage, draft
    FE-->>User: 展示草稿卡片、确认写入、继续修改、放弃
```

## 前后端契约重点

- `clientMessageId` 必须持久化，重复提交返回第一次处理结果。
- 语音 MVP 不长期保存音频，只保存 `asrText` 到 `RECORD_MESSAGE`。
- AI 无法理解时可以只返回追问消息，不生成新 `RECORD_DRAFT`。
- 生成新草稿时旧草稿必须标记为 `replaced`，`RECORD_SESSION.current_draft_id` 指向最新草稿。
