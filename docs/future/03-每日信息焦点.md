# 每日信息焦点

## 目标与边界

手机端首页保留一张共享「今日热门」入口。抽屉内按 AI、中国大事、新闻、开源四类展示，每类最多 5 条；可在抽屉中查看当天或前一天。所有用户读取同一份内容，本期不做阅读偏好、用户习惯或自动趋势分析。

只读取 RSS/Atom 中已有的标题、摘要和发布时间。系统不抓文章正文、不嵌入原网页、不调用付费检索或社交网页抓取；原站只以新窗口外链打开。没有 Horizon 服务或算法：24/72 小时时间窗、规则分类、去重与两日读取已经覆盖本期需求。

## 模块边界与来源

| 边界 | 位置 | 职责 |
|---|---|---|
| 固定 OPML 快照、RSS 请求、分类和 LLM 中文编辑 | `third/news_focus/` | 固定提交导入到 third Redis，单源限流与退避，四类候选筛选；不碰业务库。 |
| 内部生成接口 | `third_two/news_focus_router.py` | `POST /internal/news-focus/generate` 和显式目录刷新接口，仅 Docker 内部使用。 |
| 定时、业务数据和两日读取 | `backend/.../newsfocus/` | 每天 07:30 调用 third、保存共享结果和 7 日去重指纹。 |
| 首页入口与抽屉 | `frontend/src/features/newsfocus/` | 卡片、今日/昨日切换、四类连续分段展示和原文外链。 |

来源固定为 `awesome-rss-feeds-list@2ab3b9b...` 与 `awesome-rss-feeds@3a7a9e...` 的 OPML 范围；中国大事额外使用受版本控制的 `china-focus@2026-07-14-v2` 白名单（中国新闻网国内、财经及中国日报 China News）。首次导入或显式刷新时下载固定提交并合并白名单，日常生成只读 Redis 快照。白名单来源必须提供可解析的发布时间；没有发布时间的 RSS 不会接入。规则拒绝付费墙、社交转 RSS、RSSHub、`xgo`、`wechat2rss`、播客与生活方式来源。

## 生成、数据与接口

每源最多取 3 条，单源 6 秒超时、16 并发；连续失败源在 third Redis 退避。候选全局按 URL 和标题去重，再排除后端传入的此前 7 日展示指纹；同日覆盖生成不排除当天已有指纹，保证重试仍能补齐分类。每类优先近 24 小时，不足 5 条时从近 72 小时补充；每类最多 20 条交给 LLM。中国大事仅由该白名单路由，优先国家层面的政策、宏观经济、科技产业、公共事件和重要民生进展；其他来源中的政治候选不进入任何分类。模型只返回中文标题、摘要、标签、主题键和顺序，取消数值评分。

每日 07:30 使用独立的 `newsFocusTaskScheduler`，不与记录工作流轮询共用线程。需要受控补跑时，仅临时设置 `BACKEND_NEWS_FOCUS_RUN_ON_STARTUP=true` 启动一个无端口后端容器；它只生成一次，完成后立即停止，常规服务保持该开关为 `false`。

Flyway `V10__news_focus_groups_and_dedupe.sql`：

| 表 | 作用 |
|---|---|
| `NEWS_FOCUS_RUN` | 一天一次的共享生成状态、统计和错误摘要。 |
| `NEWS_FOCUS_ITEM` | 属于某日的分类键、分类内排名、中文内容、标签、来源链接和发布时间。 |
| `NEWS_FOCUS_SEEN` | 7 日标题指纹去重记忆，不供页面展示。 |

展示数据只保留当天和前一天；成功生成后清理更早的 run/item，去重指纹在 7 天后清理。当天失败时首页最多回退昨天；请求昨天时不做跨日回退。

`GET /api/user/home` 的 `newsFocus.groups` 返回四类分组。新增已登录 `GET /api/user/news-focus?date=YYYY-MM-DD`，只接受上海时区当天或前一天：

```json
{
  "available": true,
  "focusDate": "2026-07-14",
  "groups": [{
    "key": "ai",
    "title": "AI",
    "items": [{
      "rank": 1,
      "title": "中文标题",
      "summary": "只基于 RSS 元数据整理的中文摘要。",
      "tags": ["模型"],
      "source": "公开 RSS",
      "sourceUrl": "https://example.com/article",
      "publishedAt": "2026-07-14T02:00:00Z"
    }]
  }]
}
```

## 排查入口

1. 看 `NEWS_FOCUS_RUN`、`NEWS_FOCUS_ITEM` 和 `NEWS_FOCUS_SEEN` 的日期、状态、分类和错误摘要。
2. 看 `third-two-api` 的生成日志与 third Redis 的 `news-focus:catalog:v2`、`news-focus:source-health:*`；目录更新只能走内部 refresh 接口。
3. 生成失败时确认 RSS 源错误和既有 third LLM 出口；不要通过抓取原文、付费搜索或新增浏览器端请求绕过。
4. 页面异常时看 `DailyFocusCard.tsx`、`features/record/api.ts`、`shared/types/api.ts` 和 `/api/user/news-focus` 响应。
