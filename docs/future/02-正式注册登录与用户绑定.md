# 正式注册登录与用户绑定

## 目标

把当前登录从固定开发账号改为正式数据库账号、密码哈希和登录会话；保留旧的 `person_user`、`person_partner`、`person_admin` 及其历史数据。普通用户可注册登录，并通过邀请确认流程绑定另一个用户。

## 接入模块

| 层 | 位置 | 说明 |
|---|---|---|
| 前端主模块 | `frontend/src/app`、`frontend/src/shared/api` | 登录、注册、会话恢复、退出和手机端绑定入口。 |
| 后端主模块 | `backend/src/main/java/com/formygirl/identity` | 账号、密码、登录会话、admin/旧用户 bootstrap。 |
| 协作模块 | `relationship` | 绑定邀请、接受、拒绝、取消，以及绑定视角权限。 |

## 用户和权限

- 使用者：普通 `USER` 可注册；`admin` 保留为 `OPS_ADMIN`，但必须用密码登录。
- 是否需要绑定用户：只有绑定管理员视角、评论、给对方加奖品、查看绑定用户记录需要 active 绑定。
- 对方是否可操作：对方接受邀请后，双方都拥有现有默认能力 `record.comment` 和 `reward.manage`。
- 后台人员是否可操作：`OPS_ADMIN` 只进后台，不参与普通用户绑定。

## 主链路

```text
注册/登录
  -> 前端登录页
  -> /api/auth/register 或 /api/auth/login
  -> identity 校验账号密码并创建 APP_LOGIN_SESSION
  -> 前端保存 session token
  -> /api/auth/me 恢复当前用户和绑定视角

绑定用户
  -> Profile 输入对方账号
  -> relationship 创建 pending USER_BINDING
  -> 对方在 Profile 接受
  -> 后端激活双向 USER_BINDING 并补齐 USER_PERMISSION
```

## 前端影响

- 页面或组件：`LoginScreen` 增加登录/注册切换；`RoleScreen` 增加绑定用户、邀请列表和退出登录。
- API helper：`client.ts` 使用当前登录会话 token；`relationship/api.ts` 增加邀请相关接口。
- DTO 类型：`BindingInfo` 增加 incoming/outgoing 邀请；`Person` 可带 `loginName`。
- 状态和错误展示：无 active 绑定时隐藏绑定管理员能力，并在 Profile 提示先绑定用户。

## 后端影响

- Controller：`IdentityController` 增加 register/logout；`RelationshipController` 增加邀请处理接口。
- Service：`IdentityService` 改为数据库账号登录；`RelationshipService` 处理 pending/active 绑定。
- Repository：`IdentityRepository` 读写账号和 session；`RelationshipRepository` 复用 `USER_BINDING.status` 表达邀请状态。
- 需要复用的模块：`APP_PERSON` 保持用户资料和角色；旧业务数据继续按 person id 关联。
- 不能耦合的模块：`third` 不参与登录注册；飞书配置继续跟业务用户 id 绑定。

## 数据库影响

- 新增表：`APP_ACCOUNT`、`APP_LOGIN_SESSION`。
- 变更表：不改旧 person id；`USER_BINDING` 继续使用 `status` 保存 `pending`、`active`、`declined`、`cancelled`。
- 关键索引：账号名唯一、person 账号唯一、session token hash 唯一。
- 幂等或唯一约束：绑定仍由 `(requester_user_id, target_user_id)` 保证单方向唯一。

## 外部服务影响

- 是否调用 third：否。
- 是否写飞书：否。
- 是否需要保留 payload 或错误：只保留登录 session 和绑定状态，不记录密码明文。
- 是否需要 trace：第一版不进入 third trace。

## 接口

| 方法 | 路径 | 用途 |
|---|---|---|
| POST | `/api/auth/register` | 注册普通用户并返回登录态。 |
| POST | `/api/auth/login` | 账号密码登录。 |
| POST | `/api/auth/logout` | 撤销当前登录 session。 |
| GET | `/api/auth/me` | 读取当前登录人、视角和绑定状态。 |
| POST | `/api/relationship/invitations` | 发起绑定邀请。 |
| POST | `/api/relationship/invitations/{id}/accept` | 接受绑定邀请。 |
| POST | `/api/relationship/invitations/{id}/reject` | 拒绝绑定邀请。 |
| POST | `/api/relationship/invitations/{id}/cancel` | 取消自己发出的邀请。 |

## 排查入口

- 前端先看：`frontend/src/shared/api/client.ts`、`frontend/src/app/LoginScreen.tsx`、`frontend/src/pages/RoleScreen.tsx`。
- 后端先看：`identity` 和 `relationship` 包。
- 数据库先看：`APP_ACCOUNT`、`APP_LOGIN_SESSION`、`USER_BINDING`、`USER_PERMISSION`。
- 关键 ID：`person_id`、`account_id`、`binding_id`、session `token_hash`。
