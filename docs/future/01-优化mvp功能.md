# 01-优化 MVP 功能

## 目标

本方案用于记录 01-优化 MVP 的功能范围和实现口径。它先作为审核文档创建，本次已按文档落地核心前后端能力，并同步回写接口、数据库和 Codex 入口文档。

本次 MVP 以 `docs/ui` 为第一事实源：这是一个手机端双向绑定记录 App。登录后，用户可以在自己的「用户视角」和绑定对象的「管理员视角」之间切换。这里的“管理员”指绑定关系里的对方管理视角，不指独立后台运维人员。

已确认的实现口径：

- 开发数据保留两个普通用户和一个独立后台人员：`person_user`、`person_partner` 都是 `USER`，`person_admin` 是 `OPS_ADMIN`。
- 手机端管理员视角由普通用户切换产生，不使用 `/api/admin/*` 的后台运维能力。
- 记录写入沿用现有草稿确认链路：`session -> message -> draft -> confirm -> DAILY_RECORD / RECORD_DISPLAY / FEISHU_SYNC`。
- 签到每天一次，固定增加 10 积分。
- 绑定用户保存记录打分后产生记录积分：本地 AI 分数和绑定用户打分各占 50%，折算为十分制并向下取整。
- 每日积分上限为 `当天记录事项数 * 10 + 签到 10`；重复改分只按差额调整，不能重复刷分。
- 奖品先内置默认项，绑定管理员视角可以给对方添加奖品。
- 用户兑换奖品后扣积分、生成兑换记录、奖品从可兑换列表消失；绑定用户需要看到奖品已兑换提示。

## 范围原则

- UI 优先于旧接口文档。
- 旧 `docs/接口文档.md` 只保留支撑当前 UI 的记录闭环能力。
- 后台 PC 管理台、全量记录追踪、飞书重试、重写入、测试后台不进入本次 MVP。
- 评论、打分、积分、奖品只落本地业务库，不写飞书。
- 评论、打分、积分、奖品和兑换记录已作为本次本地业务能力实现。

## 接入模块

| 模块 | 职责 | 本次 MVP 用途 |
|---|---|---|
| `identity` | 登录、当前用户、视角信息 | 登录页、当前用户、用户视角和管理员视角切换。 |
| `relationship` | 双向绑定、授权关系 | 判断当前用户能否作为对方管理员查看记录、添加奖品、评论和打分。 |
| `record` | 对话记录、最近记录、展示字段 | 对话界面、用户最近记录、管理员查看绑定用户最近记录。 |
| `points` | 签到积分、记录打分积分、积分余额、奖品、兑换记录 | 首页签到加积分、积分展示、奖品列表、管理员添加奖品、用户兑换奖品、绑定用户查看兑换提示。 |
| `comment` | 记录评论和打分 | 管理员给绑定用户记录写评论和打分；用户最近记录展示评论。 |
| `style` | 移动端背景和风格 | 以当前 UI 背景和模糊玻璃效果为准，后续再扩展可配置风格。 |

## UI 页面到能力

| 页面 | 当前 UI 内容 | 后端能力 |
|---|---|---|
| 登录页 | 账号、密码、验证码、进入按钮 | `identity` 登录；验证码 MVP 可先做本地校验或占位。 |
| 首页 | 头像入口、角色提示、签到、两个底部图标 | `identity` 当前视角；`points` 签到和积分；根据视角返回入口状态。 |
| 用户/角色界面 | 头像、名称、角色切换、积分、可兑换奖品 | `identity` 视角切换；`relationship` 绑定信息；`points` 积分和奖品。 |
| 对话界面 | 历史对话、输入框、录音按钮、发送按钮 | `record` 创建会话、发送文本或语音、生成草稿或记录。 |
| 最近记录（用户视角） | 字段选择、日期/内容/打分、绑定管理员评论 | `record` 最近记录；`comment` 评论读取。 |
| 积分奖品（管理员视角） | 绑定用户积分、奖品名称、所需积分、添加奖品、奖品列表 | `relationship` 授权校验；`points` 奖品添加和列表。 |
| 最近记录（管理员视角） | 绑定用户记录、评论输入、打分输入、保存提示 | `relationship` 授权校验；`record` 记录读取；`comment` 评论和打分保存。 |

## 主链路

```text
登录
  -> 获取当前用户和绑定关系
  -> 默认进入用户视角首页
  -> 用户可签到、查看积分、进入对话、查看最近记录
  -> 用户可切换到绑定管理员视角
  -> 管理员视角可给绑定用户添加奖品
  -> 管理员视角可查看绑定用户记录并保存评论和打分
  -> 评论打分触发记录积分差额入账
  -> 用户视角最近记录展示评论和打分
  -> 用户可兑换奖品并生成兑换记录
  -> 绑定管理员视角可看到奖品被兑换提示
```

## 当前接口

以下接口已同步到 `docs/接口文档.md` 的 “01-优化 MVP 手机端绑定接口”。

| 方法 | 路径 | 模块 | 用途 |
|---|---|---|---|
| `POST` | `/api/auth/login` | `identity` | 登录并返回当前用户。 |
| `GET` | `/api/auth/me` | `identity` | 获取当前用户、当前视角和绑定状态。 |
| `POST` | `/api/identity/view-role` | `identity` | 在用户视角和绑定管理员视角之间切换。 |
| `GET` | `/api/relationship/binding` | `relationship` | 获取当前绑定对象和可操作权限。 |
| `POST` | `/api/points/checkins` | `points` | 签到并增加积分。 |
| `GET` | `/api/points/summary` | `points` | 获取当前积分、签到状态和可兑换奖品摘要。 |
| `GET` | `/api/rewards` | `points` | 查询当前视角可见奖品列表。 |
| `POST` | `/api/rewards` | `points` | 管理员视角给绑定用户添加奖品。 |
| `POST` | `/api/rewards/{rewardId}/redeem` | `points` | 用户兑换自己的可用奖品。 |
| `GET` | `/api/reward-redemptions` | `points` | 查询当前视角下的兑换记录和绑定用户提示。 |
| `POST` | `/api/record-sessions` | `record` | 创建记录对话会话。 |
| `POST` | `/api/record-sessions/{sessionId}/messages` | `record` | 发送文本消息或修改指令。 |
| `POST` | `/api/record-sessions/{sessionId}/voice-messages` | `record` | 发送语音消息；MVP 可先占位。 |
| `GET` | `/api/records/recent` | `record` | 用户视角查询自己的最近记录。 |
| `GET` | `/api/bound-user/records/recent` | `record` | 管理员视角查询绑定用户最近记录。 |
| `POST` | `/api/records/{recordId}/comment` | `comment` | 管理员视角保存评论和打分。 |

## 当前数据边界

V3 预留绑定、评论、积分、奖品和兑换表；V4 增量补充当前视角、双向绑定种子、评论打分、积分幂等来源、内置奖品和兑换提示所需字段。

| 模块 | 数据 |
|---|---|
| `identity` | 用户账号、显示名、当前视角。 |
| `relationship` | 用户绑定关系、绑定状态、授权能力。 |
| `record` | 记录会话、消息、草稿、正式记录、展示字段。 |
| `points` | 积分账户、签到流水、记录打分流水、奖品、兑换记录。 |
| `comment` | 记录评论、绑定用户打分、评论作者、更新时间。 |
| `style` | 当前 UI 风格配置；MVP 可先不入库。 |

## 旧接口收敛

| 旧能力 | 本次处理 |
|---|---|
| 用户首页、记录会话、最近记录 | 保留并按当前手机 UI 调整返回内容。 |
| 管理员每日内容配置 | 不进入本次 MVP，后续如需要应归入 `style` 或独立 future 文档。 |
| 管理员全量记录列表、追踪、重试飞书同步 | 不进入绑定管理员视角；保留给独立后台运维，不参与本次 UI。 |
| 飞书 payload、同步失败重试、重写入 | 记录确认链路仍可保留基础同步，但 UI 中不暴露后台重试能力。 |
| 评论、打分、积分、奖品、兑换记录 | 本次实现为本地业务能力，不写飞书。 |

## 前端影响

- 以 `docs/ui` 的页面结构为准重做正式前端页面。
- 登录后默认显示手机端首页，不使用 PC 管理台布局。
- 当前视角决定首页底部两个图标入口：
  - 用户视角：对话、最近记录。
  - 管理员视角：积分奖品、绑定用户最近记录。
- 积分、奖品、评论、打分、兑换记录按本文件确认接口实现。

## 后端影响

- `identity` 负责当前登录用户和当前视角。
- `relationship` 负责绑定关系和管理员视角授权。
- `record` 只负责记录主链路和展示，不承载积分奖品逻辑。
- `points` 负责签到积分和奖品。
- `comment` 负责记录评论和打分。
- `style` 只负责背景和风格，不影响记录同步。

## 排查入口

| 问题 | 先看 |
|---|---|
| 登录或视角不对 | `identity` |
| 不能切换管理员视角 | `relationship` |
| 签到积分没增加 | `points` |
| 打分后积分没变化 | `comment` + `points` |
| 奖品添加失败 | `points` + `relationship` |
| 奖品兑换后仍显示 | `points` |
| 绑定用户看不到兑换提示 | `points` + `relationship` |
| 对话或最近记录异常 | `record` |
| 评论或打分没显示 | `comment` + `record` |
| 背景和页面风格异常 | `style` |

## 本次暂缓项

- 验证码 MVP 只保留 UI，不做真实校验。
- 对话界面本次沿用完整 AI 草稿确认流程。
- 语音输入本次只保留入口，不实现真实 ASR。
- 风格背景本次只按当前 UI 固定背景和玻璃层实现，不做独立配置页。
