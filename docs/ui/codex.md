# 双向绑定记录 App UI 原型说明

## 产品定位

这是一个双向绑定的手机端 Web 应用。两个用户可以互相绑定，每个人既可以作为自己的「用户」使用记录、签到、查看最近内容，也可以切换成对方的「管理员」，给绑定用户添加奖品、查看记录、评论和打分。

## 页面结构

### 登录页

登录页只保留基础账号、密码、验证码和进入按钮。登录页不是本次视觉重点。

### 首页

首页是完整背景页面，不使用传统卡片式后台布局。

包含：

- 左上角用户头像，点击进入用户界面。
- 右上角当前角色提示。
- 中间签到组件。
- 点击签到后增加积分，并显示一句反馈文案。
- 左下角和右下角为图标入口，不显示文字。

首页底部图标随当前角色变化：

- 用户视角：左下角为对话，右下角为最近。
- 管理员视角：左下角为积分奖品，右下角为最近。

### 用户界面 / 角色界面

点击左上角头像进入。背景保持首页背景，但界面层使用模糊玻璃效果。

包含：

- 顶部中间显示头像、名称和当前角色。
- 点击顶部身份区域可在用户视角和管理员视角之间切换。
- 中间展示积分。
- 下方展示可兑换奖品。
- 用户视角可兑换奖品；兑换后奖品从列表消失。
- 右上角返回首页。返回后首页底部图标根据当前角色变化。

### 对话界面

用户视角进入。

包含：

- 简洁顶部栏。
- 历史对话记录。
- 底部输入框。
- 录音按钮。
- 发送按钮。

### 最近记录界面（用户视角）

展示自己的最近记录。

包含：

- 顶部字段选择区。
- 默认展示日期、内容、打分三个字段。
- 每条记录下面显示绑定管理员的评论。

### 积分奖品界面（管理员视角）

管理员给绑定用户添加可兑换奖品。

包含：

- 绑定用户当前积分展示。
- 奖品名称输入。
- 所需积分输入。
- 添加奖品按钮。
- 已有奖品列表。
- 绑定用户奖品被兑换后的提示列表。

### 最近记录界面（管理员视角）

管理员查看绑定用户的最近记录。

包含：

- 时间、内容、打分三个字段。
- 每条记录后面直接提供评论输入和打分输入。
- 保存成功显示成功提示。
- 输入不合法显示失败提示。

## 视觉原则

- 整体以手机端为核心。
- 首页使用完整背景。
- 功能页保留背景，但增加模糊玻璃层。
- 不使用 PC 管理台样式。
- 不使用复杂导航文字。
- 底部入口尽量图标化。
- 记录和管理能力都整合到同一个移动端应用里。
