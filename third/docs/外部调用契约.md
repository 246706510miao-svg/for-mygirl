# third Workflow v1 外部调用契约

本文档给 Spring Boot 和其他外部业务后端使用。`third` 与 `third_two` 共用同一套 Pydantic v1 模型，稳定字段统一使用 camelCase；开放 JSON 只允许出现在 metadata、preview 和 Tool 输出等明确叶子节点。

旧的无版本 `/workflows/...` 与 `/internal/workflows/...` 接口只用于第一阶段兼容，不再允许新增调用方。新代码必须使用 `/v1/...`。
旧入口每次调用都会记录 `deprecated third workflow route used`；生产连续 24 小时无此日志后，进入第二阶段删除旧路由。

## 状态机

```text
invoke: POST /v1/workflows/invoke
status/snapshot: GET /v1/workflows/{sessionId} + GET /v1/workflows/{sessionId}/snapshot
waiting_user: 后端返回 confirmation 给前端
resume: POST /v1/workflows/{sessionId}/resume
final/snapshot: GET /v1/workflows/{sessionId} + GET /v1/workflows/{sessionId}/snapshot
```

`/debug` 只是托管了同一套状态机的调试页面，不是另一条业务流程。

## invoke 输入

```json
{
  "content": [
    {
      "text": "今天学习了韩语，写到飞书里面去"
    }
  ],
  "metadata": {
    "businessSessionId": "session_xxx",
    "idempotencyKey": "cmid_xxx"
  }
}
```

字段规则：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `content[0].text` | 是 | 用户原始输入。业务后端不要添加“生成草稿：”等固定前缀。 |
| `metadata.businessSessionId` | 是 | 外部业务会话 ID，例如 SpringBoot `RECORD_SESSION.id`。 |
| `metadata.idempotencyKey` | 是 | 外部幂等 ID，消息阶段使用 `clientMessageId`。 |
| `metadata.businessRecordId` | 可选 | 已有正式记录 ID，重试或追踪时使用。 |
| `metadata.operation` | 可选 | 只用于显式 override；默认不传，由 workflowagent 决定。 |

## 轻量状态接口

`POST /v1/workflows/invoke`、`GET /v1/workflows/{sessionId}` 和 resume 返回同一轻量结构：

```json
{
  "sessionId": "sess_xxx",
  "status": "queued | running | waiting_user | success | failed | cancelled",
  "content": [
    {
      "text": "最终文本答案"
    }
  ],
  "errorText": null
}
```

轻量响应只负责状态、最终文本和错误。confirmation、decision 和 outputs 只能从 snapshot 读取，禁止在两个响应之间兼容猜字段。

## outputs 字段

`outputs.draft` 来自 `artifact_key=record.draft`：

```json
{
  "title": "今日自律记录",
  "recordDate": "2026-06-22",
  "summary": "今天学习了韩语",
  "score": 80,
  "tags": ["学习"],
  "suggestion": "明天继续保持。",
  "previewText": "今天学习了韩语",
  "source": "llm"

```

`outputs.writePayload` 来自 `artifact_key=validation.write_payload`：

```json
  "request_key": "create_request",

    "original_input": "今天学习了韩语，写到飞书里面去",

            "事项名称": "韩语学习",
            "总结": "今天学习了韩语"
          }
        }
      ]
    }
  },
  "idempotency_key": "create_record:feishu_bitable:hash",
  "payload_hash": "hash",
  "warnings": [],
  "preview": {
    "record_count": 1,
    "records": [
      {
        "record_id": null,
        "fields": {
          "事项名称": "韩语学习",
          "总结": "今天学习了韩语"
        },
        "lookup": {}
      }
    ]
  },
  "expires_at": "2026-06-29T10:00:05"
}
```

`outputs.writeResult` 来自 `artifact_key=write_result`，内容由具体 Tool 决定，后端必须整体保存到同步追踪数据中。

`outputs.tableSchema` 来自 `feishu.table_schema_after` 或 `feishu.table_schema`，用于后端排查字段对齐问题。

`outputs.records` 来自 `feishu.records` 或 `feishu.candidate_records`，用于查询、更新、删除类 workflow 的候选记录展示或追踪。

`outputs.finalAnswer` 来自 `artifact_key=final.answer`。

## 后端消费规则

- 后端必须保存 `businessSessionId <-> third sessionId` 映射。
- 后端必须读取 snapshot，而不是直接猜 artifact key。
- 后端可以读取 `outputs.writePayload.preview` 生成用户确认预览。
- 后端可以读取 `outputs.draft` 或 `outputs.writePayload.preview` 生成本地 `RECORD_DRAFT`。
- 后端给前端的字段必须是后端选择后的稳定字段；third 原始 JSON 只保存到 trace/sync payload，不直接透给前端。
- `status=waiting_user` 时，后端必须把 `thirdSessionId` 和 `confirmationId` 返回给前端。
- `status=waiting_user` 时 snapshot 必须存在非空的 `confirmationId`、`requestText` 和合法 `interactionKind`；缺失即为 `THIRD_CONTRACT_MISMATCH`，不得读取其他字段名兜底。
- 用户确认时，后端必须调用 `/v1/workflows/{sessionId}/resume`，不得绕过 third 确认门。
- v1 可以增加新字段；调用方必须忽略未知字段。删除、改名必填字段或新增未约定枚举值属于破坏性变更，必须发布新版本。
- `outputs.*`、`confirmation.preview`、`Artifact.data/schema` 是开放 JSON 叶子；session、confirmation、decision、artifact 外壳不是开放 Map。

## resume 输入

```json
{
  "confirmationId": "confirm_xxx",
  "approved": true,
  "response": "approve",
  "content": [
    {
      "text": "业务前端已确认执行"
    }
  ]
}
```

`response` 只能是 `approve`、`answer`、`modify`、`cancel`；`interactionKind` 只能是 `confirm`、`clarify`、`choose_candidate`。`approved=false` 且 `response=cancel` 时不会执行确认门后面的写入 Tool。

## override

默认不传 `metadata.operation`。当前只保留一个 override：

| `metadata.operation` | 行为 |
| --- | --- |
| `draft_generate` | 强制使用 `record_draft` template，输出 `outputs.draft`，不写飞书。 |
