# 前端界面组件与优化说明

本文档用于后续优化前端视觉和交互时对齐页面能力。内容以当前 `frontend/src` React 实现为准，旧 `docs/ui` 原型只作为页面口径参考。

## 页面清单

| 页面 | 入口和角色 | 必须实现的功能 | 当前可抽象组件 | 可优化方向 |
|---|---|---|---|---|
| 登录页 | 未登录用户进入应用 | 输入账号、密码、本地验证码；验证码可刷新；登录中禁用提交；展示本地校验或接口错误 | 登录面板、表单字段、验证码行、主按钮、状态提示 | 品牌首屏、输入框层级、验证码样式、错误提示位置 |
| 手机端首页 | 普通用户或绑定管理员视角 | 展示头像入口、当前视角、首页文案、积分或签到状态；用户视角可签到；绑定管理员视角可进入奖品管理；底部按视角切换入口 | 移动端背景壳、头像按钮、角色标签、签到/积分组件、底部图标导航、Toast | 背景质感、中心组件形态、图标语义、首页留白、点击反馈 |
| 用户/角色页 | 首页左上角头像进入 | 展示当前身份、当前视角、积分、奖品列表；可在用户视角和绑定管理员视角之间切换；用户视角可兑换奖品 | 玻璃层页面、返回按钮、身份切换块、积分指标、奖品列表、兑换按钮 | 头像和身份层级、角色切换反馈、奖品卡片、积分强调方式 |
| 对话页 | 用户视角首页左下角进入 | 展示历史对话；输入文本并发送；首次发送创建记录会话；展示 AI 草稿；确认草稿写入；语音入口暂不接真实 ASR | 二级标题栏、消息列表、消息气泡、草稿卡片、底部输入栏、语音按钮、发送按钮 | 对话气泡节奏、草稿预览层级、键盘区适配、语音入口状态 |
| 用户最近记录页 | 用户视角首页右下角进入 | 选择展示字段；默认展示日期、内容、打分；展示记录列表和管理员评论；空列表提示 | 二级标题栏、字段选择器、记录列表、记录字段组、评论摘要、空状态 | 字段选择控件、记录信息密度、评论视觉归属、列表间距 |
| 管理员积分奖品页 | 绑定管理员视角首页左下角或中心按钮进入 | 展示绑定用户积分；输入奖品名称和所需积分；添加奖品；展示已有奖品；展示兑换提示 | 二级标题栏、积分指标、奖品表单、奖品列表、兑换提示列表、Toast | 表单横向布局、奖品状态区分、兑换提示可读性、成功反馈 |
| 管理员最近记录页 | 绑定管理员视角首页右下角进入 | 展示绑定用户最近记录的时间、内容、打分；填写评论和打分；校验评论非空和 0-100 分；保存后刷新 | 二级标题栏、管理员记录卡片、记录字段组、内联评论表单、保存按钮、错误提示 | 评论输入体验、评分控件、保存状态、记录卡片层级 |
| Ops 后台页 | `OPS_ADMIN` 或 `ADMIN` 登录后进入 | 初始化后台 token；查看今日记录、待反馈、异常统计；刷新列表；查看记录详情；查看追踪 JSON | PC 后台布局、侧边栏、刷新按钮、汇总指标、记录表格、详情 JSON 面板、追踪 JSON 面板 | PC 信息密度、表格列宽、详情面板可读性、异常状态突出 |

## 通用组件抽象

| 组件 | 使用页面 | 功能边界 | 优化关注点 |
|---|---|---|---|
| `MobileAppShell` | 登录页、手机端所有页面 | 限制手机端宽度、承载背景、安全区和全局字体 | 背景图或主题纹理、最大宽度、移动端沉浸感 |
| `GlassScreen` | 用户/角色页、对话页、最近记录页、奖品页 | 二级页面容器，保留首页背景并叠加内容层 | 透明度、模糊强度、滚动边界、页面内留白 |
| `ScreenHeader` | 对话页、最近记录页、奖品页 | 返回按钮和页面标题 | 返回图标、标题尺寸、左右占位一致性 |
| `Toast` | 手机端所有异步动作 | 展示加载、成功、失败、校验提示 | 成功/失败颜色区分、出现位置、自动消失策略 |
| `EmptyState` | 对话、记录、奖品、后台列表 | 无数据时占位 | 文案层级、图标、与列表加载态区分 |
| `MetricLine` | 用户/角色页、奖品页、Ops 汇总 | 展示关键数字，例如积分或统计数量 | 数字视觉权重、单位、卡片背景 |
| `RewardList` | 用户/角色页、管理员奖品页 | 展示奖品名称、描述、所需积分和兑换状态 | 可兑换/不可兑换状态、按钮位置、奖品类型图标 |
| `RecordList` | 用户最近记录页 | 展示可配置字段和管理员评论 | 字段排布、长文本换行、评论归属 |
| `FieldSelector` | 用户最近记录页 | 控制记录列表展示字段，至少保留一个字段 | 可点区域、选中态、字段过多时的扩展方式 |
| `ChatHistory` | 对话页 | 展示用户消息、AI 消息和系统消息 | 气泡对齐、时间/状态、滚动到底部 |
| `DraftPanel` | 对话页 | 展示记录草稿标题、预览、分数和确认写入按钮 | 草稿与普通消息区分、确认按钮强调、修改入口预留 |
| `Composer` | 对话页 | 承载语音入口、文本输入和发送 | 固定底部、安全区、输入高度、禁用态 |
| `InlineCommentForm` | 管理员最近记录页 | 评论和打分输入，保存单条记录 | 评分控件、错误定位、保存后状态 |
| `OpsTable` | Ops 后台页 | 展示记录日期、摘要、状态、详情和追踪操作 | 列宽、状态色、行 hover、分页和筛选预留 |
| `JsonPanel` | Ops 后台页 | 展示详情或追踪原始 JSON | 折叠、复制、格式化、关键信息摘要 |

## 页面能力边界

### 登录页

- 登录页只负责账号、密码、本地验证码和登录状态，不负责注册、找回密码、真实验证码服务。
- 成功登录后由 `App.tsx` 根据后端角色分流：普通角色进入手机端，`OPS_ADMIN` / `ADMIN` 进入 Ops 后台。
- 可优化内容集中在首屏品牌感和表单可读性，不要改变登录分流逻辑。

### 手机端首页

- 首页是手机端的主入口，不做复杂文字导航。
- 用户视角底部入口是对话和最近记录，绑定管理员视角底部入口是积分奖品和绑定用户最近记录。
- 中心区域必须保留签到或积分主操作，因为这是首页的主要行为入口。
- 可优化内容包括背景、中心组件、底部图标、角色标签和按钮反馈。

### 用户/角色页

- 该页是身份、视角、积分和奖品的集合页。
- 身份区域点击后切换 `USER` / `BOUND_ADMIN` 视角；如果没有绑定对象，切换按钮应不可用。
- 用户视角可兑换奖品，绑定管理员视角只查看和管理绑定用户相关数据。
- 可优化内容包括身份卡片、角色切换动效、积分展示和奖品列表。

### 对话页

- 对话页负责记录写入主链路：输入内容、发送消息、生成草稿、确认写入。
- 语音按钮当前只是入口提示，不接真实语音识别。
- 草稿确认后需要刷新最近记录数据，并清空当前会话和草稿。
- 可优化内容包括聊天气泡、草稿卡片、输入栏固定、键盘适配和发送中状态。

### 用户最近记录页

- 页面默认展示日期、内容、打分，并允许用户调整展示字段。
- 记录下方需要展示管理员评论，没有评论时也要给出明确占位。
- 字段选择至少保留一个字段，避免列表为空结构。
- 可优化内容包括字段选择器、记录卡片密度、评论区域和空状态。

### 管理员积分奖品页

- 页面面向绑定管理员，用来给绑定用户添加可兑换奖品。
- 表单必须校验奖品名称和正整数积分。
- 兑换提示列表用于让管理员知道绑定用户已经兑换过什么。
- 可优化内容包括奖品录入方式、积分成本展示、兑换提示分组和成功反馈。

### 管理员最近记录页

- 页面面向绑定管理员，用来查看绑定用户最近记录并写评论和打分。
- 评论不能为空，打分范围是 0-100。
- 保存成功后刷新当前记录列表，保证最新评论和分数显示出来。
- 可优化内容包括评分输入控件、评论输入区、单条保存状态和校验错误展示。

### Ops 后台页

- Ops 是独立 PC 后台，不套手机端视觉。
- 当前功能聚焦记录排查：统计、列表、详情 JSON、追踪 JSON。
- 后续如果增加重试、重写入、筛选或分页，应继续放在 Ops 模块，不混入用户端手机页面。
- 可优化内容包括表格、筛选栏、详情抽屉、JSON 可读性和异常状态突出。

## 优化优先级

| 优先级 | 内容 | 说明 |
|---|---|---|
| P0 | 保留页面能力和角色分流 | 视觉优化不能破坏登录、视角切换、签到、记录写入、奖品、评论和 Ops 排查能力 |
| P1 | 先抽通用组件 | 优先把标题栏、Toast、空状态、指标、记录卡片、奖品卡片、表单控件统一 |
| P1 | 替换临时图标和按钮 | 首页底部、返回、语音、记录入口、奖品入口应使用一致图标体系 |
| P2 | 建立主题变量 | 背景、主色、强调色、文字色、边框、阴影、间距统一由变量控制 |
| P2 | 优化移动端细节 | 处理安全区、键盘区域、按钮触达面积、长文本换行和滚动区域 |
| P3 | 增加轻量动效 | 只给切换、Toast、按钮反馈、卡片进入增加动效，不影响核心操作 |

## 设计约束

- 手机端用户应用和 PC Ops 后台是两套界面体系，不要混用布局。
- 用户端优先保证 iPhone 宽度体验，按钮触达高度不少于 44px。
- 首页底部入口保持图标化，复杂说明放到页面内部，不放在首页导航。
- 所有异步动作都要有加载、成功或失败反馈。
- 表单校验要在当前页面给出，不依赖用户打开控制台或查看 Network。
- UI 优化只调整展示和交互，不直接改变 `shared/api`、feature `api.ts` 或后端接口契约。
